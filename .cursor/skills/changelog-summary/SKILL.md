# Changelog Summary（本项目补充规则）

与 **global-changelog-summary** 配合使用。流程与步骤以 global skill 为准；以下仅为本项目**差异规则**。

## 输出文件

- `docs/release/CHANGELOG.md`（与 global 默认一致）

## 术语一致

- 语言一致（默认中文）。
- 术语按项目现有用法：**Bookmark**、**Tag**、**Workstation** 等保持一致。

## 写作风格（条目点名）

- 如涉及 **Options / Popup / Background** 等入口，尽量在条目中点名模块。
